# api_client.py 改进行动计划

## 优先级排序的行动计划

### P0 - 紧急（安全与可用性）
1. 提取 `_request()` 私有方法，统一错误处理逻辑
2. 添加 `try/except` 包裹网络请求（捕获 ConnectionError, Timeout, HTTPError, JSONDecodeError）
3. 所有请求添加 `timeout` 参数
4. 自定义异常类体系（APIClientError, APIHTTPError, APINetworkError, APITimeoutError）

### P1 - 重要（健壮性与可维护性）
5. 实现重试机制（指数退避，可配置重试次数）
6. 使用 `logging` 替代潜在的空异常
7. 为 DELETE 返回状态信息而非 None

### P2 - 优化（开发者体验）
8. 解决 DRY 问题（统一 URL 拼接）
9. 添加 SSL 验证选项
10. 完善 docstring，标注可能抛出的异常类型
