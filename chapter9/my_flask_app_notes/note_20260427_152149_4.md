# [BLOCKER] 错误处理与异常管理不完善

## 问题描述

### 1. 裸 `except` 使用
`codebase_maintainer.py` 中存在 3 处 `except Exception` 捕获，过于宽泛：
- 第225行: `except Exception as e:` - 笔记检索失败时静默处理
- 第248行: `except Exception:` - JSON 解析失败
- 第408行: `except Exception:` - 笔记摘要获取失败

### 2. api_client.py 缺少自定义异常
- `get()`, `post()`, `put()`, `delete()` 均直接调用 `response.raise_for_status()`
- 没有自定义异常类区分网络错误、认证错误、业务错误
- 调用方无法区分不同类型的错误

### 3. data_processor.py 缺乏输入验证
- `process_data()` 没有对空数据、None 值进行校验
- `clean_data()` 直接 `dropna()` 丢弃数据，没有日志记录丢弃了多少行

## 影响
- 问题难以定位和排查
- 生产环境中可能导致数据静默丢失
- API 调用失败时错误信息不够明确

## 严重程度
🟡 中 - 影响可维护性和可观测性
